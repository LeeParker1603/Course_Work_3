# Курсовая работа: Мониторинг воздушного пространства (Sky Tracker)

## Описание проекта
Приложение на Python для сбора, обработки и хранения данных о самолетах, находящихся в воздухе в режиме реального времени. Проект демонстрирует интеграцию с внешними API, объектно-ориентированное проектирование (ООП), работу с реляционными базами данных (PostgreSQL) и автоматическое тестирование.

## Возможности

- Получение географических координат (bounding box) страны через API Nominatim.
- Получение списка самолетов в указанном регионе через API OpenSky Network.
- Сохранение данных о самолетах в PostgreSQL базу данных.

## Реализованные методы DBManager
* `create_database()` — Статический метод автоматического создания чистой базы данных и связанных таблиц с каскадным удалением.
* `get_countries_and_aeroplanes_count()` — Статистика количества воздушных судов по странам.
* `get_all_aeroplanes()` — Вывод полного списка зарегистрированных в базе самолетов.
* `get_avg_speed()` — Расчет средней скорости всех бортов в м/с.
* `get_aeroplanes_with_higher_speed()` — Фильтрация самолетов, летящих быстрее среднего значения.
* `get_aeroplanes_with_keyword()` — Поиск воздушных судов по маске позывного (например, поиск канадских бортов `ACA`).


## Требования

- Python 3.8 или выше
- Библиотека `requests` (устанавливается автоматически через `pip install -r requirements.txt`)
- Библиотека `psycopg2` (устанавливается автоматически через `pip install -r requirements.txt`)

## Установка и запуск

1. Клонируйте репозиторий или скачайте файлы проекта.
2. Перейдите в корневую директорию проекта.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

Если файла ```requirements.txt``` нет, установите requests и psycopg2 вручную:

```bash
pip install requests
pip install psycopg2
```

4. Конфигурация переменных окружения:
Создайте в корне проекта файл `.env` и укажите параметры подключения к вашей локальной СУБД PostgreSQL:
```env
DB_USER=postgres
DB_PASSWORD=ваш_пароль_от_postgres
DB_HOST=localhost
DB_PORT=5432
```

Запустите приложение:

```bash
python main.py
```

## Описание модулей

- ```src/api.py```

Содержит абстрактный класс BaseAPI и его реализацию OpenSkyAPI. Класс умеет получать bounding box страны через Nominatim и запрашивать список самолетов через OpenSky Network.


- ```src/DBclass.py```

Класс для инкапсуляции всей логики работы с базой данных (добавление, 
фильтрация, 
агрегация данных через SQL-запросы). Включает статический метод `create_database` для автоматического развертывания структуры БД.

- ```main.py```

Главный скрипт, запускающий цикл сбора данных по целевым странам.

## Тестирование

Для запуска тестов (если они реализованы) выполните:

```bash

pytest tests/
```

Тесты покрывают основные сценарии: создание объектов, валидацию, сравнение, преобразование данных, а также (при необходимости) мокирование API-запросов.

## Используемые API

**Nominatim** (https://nominatim.openstreetmap.org) – для получения географических координат страны.

**OpenSky Network** (https://opensky-network.org) – для получения информации о самолетах в реальном времени.

### Примечания

Для корректной работы с Nominatim необходимо указывать корректный User-Agent (в коде установлен CourseWorkApp/1.0).

Данные OpenSky Network могут быть неполными или отсутствовать для некоторых регионов.
