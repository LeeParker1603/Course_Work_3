# Курсовая работа: Интеграция с API для получения информации о самолетах

Данный проект представляет собой консольное приложение на Python, которое интегрируется с внешними API для получения данных о самолетах, находящихся в воздушном пространстве выбранной страны. Полученная информация сохраняется в JSON-файл и может быть отфильтрована и отсортирована по различным критериям.

## Возможности

- Получение географических координат (bounding box) страны через API Nominatim.
- Получение списка самолетов в указанном регионе через API OpenSky Network.
- Сохранение данных о самолетах в JSON-файл.
- Вывод топ N самолетов по высоте полета.
- Фильтрация самолетов по стране регистрации.
- Отображение всех полученных самолетов.

## Требования

- Python 3.8 или выше
- Библиотека `requests` (устанавливается автоматически через `pip install -r requirements.txt`)

## Установка и запуск

1. Клонируйте репозиторий или скачайте файлы проекта.
2. Перейдите в корневую директорию проекта.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

Если файла ```requirements.txt``` нет, установите requests вручную:

```bash
pip install requests
```

Запустите приложение:

```bash
python main.py
```

## Использование

При запуске программа запросит название страны (например, "Russia", "Spain", "United States").

После успешного получения данных будет показано меню с доступными действиями:

- Сохранить данные в файл (aeroplanes.json).

- Показать топ N самолетов по высоте.

- Отфильтровать самолеты по стране регистрации.

- Показать все полученные самолеты.

## Описание модулей

- ```src/api.py```

Содержит абстрактный класс BaseAPI и его реализацию OpenSkyAPI. Класс умеет получать bounding box страны через Nominatim и запрашивать список самолетов через OpenSky Network.

- ```src/aeroplane.py```

Определяет класс Aeroplane, который хранит информацию о самолете: позывной, страна регистрации, скорость, высота и другие параметры. Класс поддерживает сравнение по скорости (операторы <, >) и по высоте (методы altitude_lt, altitude_gt), а также валидацию входных данных. Статический метод from_api_data преобразует сырые данные от OpenSky в список объектов Aeroplane.

- ```src/storage.py```

Содержит абстрактный класс BaseStorage, определяющий контракт для работы с хранилищем. Реализация JSONStorage сохраняет данные в JSON-файл, обеспечивая добавление, получение по критериям и удаление записей.

- ```src/user_interface.py```

Функция user_interaction предоставляет консольное меню для взаимодействия с пользователем: сохранение данных, вывод топ-N по высоте, фильтрация по стране, вывод всех самолетов.

- ```main.py```

Главный скрипт, который создает экземпляр OpenSkyAPI, запрашивает у пользователя страну, получает данные, преобразует их в объекты Aeroplane, инициализирует хранилище и передает управление в user_interaction.

Тесты покрывают основные сценарии: создание объектов, валидацию, сравнение, преобразование данных, а также (при необходимости) мокирование API-запросов.

## Используемые API

**Nominatim** (https://nominatim.openstreetmap.org) – для получения географических координат страны.

**OpenSky Network** (https://opensky-network.org) – для получения информации о самолетах в реальном времени.

### Примечания

Для корректной работы с Nominatim необходимо указывать корректный User-Agent (в коде установлен CourseWorkApp/1.0).

Данные OpenSky Network могут быть неполными или отсутствовать для некоторых регионов.

При сохранении в JSON-файл дубликаты (по icao24) не создаются.
